Error Card

Shown when a user submission fails to resolve — timeout, upstream outage, transient network failure. Echoes the user's prompt back so the retry is frictionless, and surfaces the reassurance that the round hasn't been counted.

Timeout — retryable
Transient failure

The most common case. Body copy frames the failure as almost always transient, confirms no round was consumed, and offers both a direct retry and an edit path in case the user wants to rephrase.

Consultation interrupted
The spell failed to resolve.
"Give me spells for fighting a dragon at sea."
The oracle didn't respond in time. This is almost always a transient network hiccup — try again. The round hasn't been counted against your five.

Upstream unavailable
Service down

When we know the backend is degraded, body copy shifts to set expectations — the retry is still available but may not help immediately.

Consultation interrupted
The oracle is unreachable.
"Concentration-free 3rd-level damage spells for a sorcerer."
The recommendation service is currently unavailable. This isn't your submission — it's on our side. Try again in a minute. The round hasn't been counted against your five.
error: upstream_503
Validation — submission rejected
Client-side reject

Rare: the query was rejected before reaching the oracle (e.g. empty, too long, or otherwise malformed). Only the edit path is offered; retry as-submitted wouldn't help.
